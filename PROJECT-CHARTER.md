# Project Charter

## Mission
Build Sentinel as a flagship open-source project.

## Working Agreement
- Documentation is the source of truth.
- Before implementing a subsystem:
  1. Review SDS.
  2. Improve docs if required.
  3. Implement.
  4. Add tests.
  5. Update documentation.
  6. Update TASKS.md.
- Never stop after one completed task.
- Continuously pick the highest-priority unblocked task.
- Pause only when:
  - architecture is ambiguous,
  - credentials/infrastructure are required,
  - multiple major tradeoffs exist,
  - or an action risks data loss/security.
- Create ADRs for significant decisions.
- Research existing OSS projects for concepts only; never copy code.
